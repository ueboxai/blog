UE 资产导入管道优化：从 FBX 到引擎的自动化工作流

2026年3月24日 13 分钟阅读 虚幻盒子团队

资产导入是 UE 项目中最耗时且容易出错的环节之一。美术在 Maya/Blender/3ds Max 中制作的资产，导入 UE 时常常出现缩放错误、材质丢失、碰撞体问题。本文介绍如何构建自动化的资产导入管道，将导入时间从小时级缩短到分钟级。

一、FBX 导入常见问题分析

1.1 缩放与单位问题

最常见的导入问题，根源在于 DCC 工具与 UE 的单位系统差异：

DCC 工具 默认单位 导入 UE 表现
Maya 厘米 (cm) ✅ 正确
3ds Max 英寸/通用单位 ⚠️ 需要转换
Blender 米 (m) ⚠️ 需要 ×100
Cinema 4D 厘米 (cm) ✅ 正确

1.2 坐标系差异

不同软件的坐标系定义不同：

  • Maya/3ds Max：Y-up（导入 UE 需要转换）
  • Blender：Z-up（与 UE 一致）
  • UE：Z-up，左手坐标系

1.3 材质与纹理丢失

FBX 导入时材质不会自动关联纹理，需要手动重新指定。常见原因：

  • 纹理路径是绝对路径，在其他电脑上无法找到
  • 材质名称不匹配
  • 使用了 UE 不支持的材质节点

二、DCC 工具导出规范

2.1 Maya 导出设置

; FBX 导出选项（Maya）
- File Type: FBX export
- Current Selection: Yes（只导出选中的）
- Animation: 根据需要
- Cameras: No（除非需要）
- Lights: No（除非需要）
- Embed Media: Yes（嵌入纹理）
- Axis Conversion: Y-up（自动转换）
- Units: Centimeters
- Up Axis: Y
- Geometry:
  - Smoothing Groups: Yes
  - Triangulate: No（让 UE 处理）
  - Tangents/Binormals: Yes

2.2 Blender 导出设置

; FBX 导出选项（Blender）
- Path Mode: Copy（复制纹理）
- Embed Textures: Yes
- Scale: 0.01（米转厘米）
- Apply Transform: Yes
- Forward: -Z
- Up: Y（Blender 是 Z-up，需要转换）

2.3 命名与层级规范

导出前确保：

  • 对象名称符合项目命名规范
  • 删除空组和历史节点
  • 冻结变换（Freeze Transformations）
  • 删除构造历史（Delete History/Modifier Stack）

三、UE 导入设置优化

3.1 静态网格导入设置

; 推荐的 FBX Import Options
- Mesh:
  - Transform Vertex to Absolute: Yes
  - Bake Pivot in Vertex: No
  - Import Mesh LODs: Yes（如果 FBX 包含 LOD）
  - Normal Import Method: Import Normals and Tangents
  - Combine Meshes: No（保持独立对象）

- Materials:
  - Import Materials: No（使用项目材质模板）
  - Import Textures: No（单独导入纹理）

- Collision:
  - Auto Generate Collision: No（手动设置更精确）
  - Collision Complexity: CTF_UseDefault

- LOD:
  - Generate LODs: Yes（如果没有预计算 LOD）
  - Number of LODs: 4
  - LOD0 Triangle Percent: 100%
  - LOD1 Triangle Percent: 50%
  - LOD2 Triangle Percent: 25%
  - LOD3 Triangle Percent: 12%

3.2 骨骼网格导入设置

; Skeletal Mesh Import Options
- Mesh:
  - Import Mesh: Yes
  - Import Morph Targets: Yes（如果有 blend shapes）
  - Update Skeleton Reference Pose: Yes

- Skeleton:
  - Skeleton: 选择已有骨架或创建新的
  - Use T0 as Reference Pose: Yes

- Animation:
  - Import Animations: Yes
  - Animation Length: Exported Time
  - Import Meshes in Bone Hierarchy: Yes

四、自动化导入管道构建

4.1 Python 自动化脚本

使用 UE Python API 批量导入：

# import_assets.py
import unreal

def import_fbx(fbx_path, destination_path):
    """导入单个 FBX 文件"""
    
    # 设置导入选项
    options = unreal.FbxImportUI()
    options.set_editor_property('import_mesh', True)
    options.set_editor_property('import_materials', False)
    options.set_editor_property('import_textures', False)
    options.set_editor_property('import_animations', False)
    
    # 静态网格设置
    options.static_mesh_import_data.set_editor_property(
        'normal_import_method', 
        unreal.FBXNormalImportMethod.FBXNIM_IMPORT_NORMALS_AND_TANGENTS
    )
    
    # 执行导入
    task = unreal.AssetImportTask()
    task.set_editor_property('filename', fbx_path)
    task.set_editor_property('destination_path', destination_path)
    task.set_editor_property('options', options)
    task.set_editor_property('automated', True)
    
    unreal.AssetToolsHelpers.get_asset_tools().import_asset_tasks([task])
    return task

# 批量导入
fbx_files = unreal.EditorUtilityLibrary.get_selected_assets()
for fbx in fbx_files:
    import_fbx(fbx.get_path_name(), '/Game/Imported')

六、碰撞体自动生成

6.1 碰撞体类型选择

  • Box：建筑、箱子、简单几何体
  • Sphere/Capsule：角色、圆形物体
  • Convex：复杂但凸的物体
  • Complex：精确碰撞，性能开销大

6.2 自动碰撞生成脚本

# generate_collision.py
import unreal

def generate_collision(static_mesh, complexity):
    """为静态网格生成碰撞体"""
    
    # 获取静态网格编辑器子系统
    editor_subsystem = unreal.get_editor_subsystem(
        unreal.StaticMeshEditorSubsystem
    )
    
    # 根据复杂度选择算法
    if complexity == "box":
        editor_subsystem.set_generate_box_collision(static_mesh, True)
    elif complexity == "convex":
        editor_subsystem.set_generate_convex_collision(
            static_mesh, 
            hull_count=4,
            max_hull_verts=16
        )
    elif complexity == "complex":
        body_setup = static_mesh.get_body_setup()
        body_setup.set_editor_property('collision_trace_flag', 
            unreal.CollisionTraceFlag.CTF_USE_COMPLEX_AS_SIMPLE)
    
    # 保存修改
    unreal.EditorAssetLibrary.save_loaded_asset(static_mesh)

# 批量处理
assets = unreal.EditorUtilityLibrary.get_selected_assets()
for asset in assets:
    if isinstance(asset, unreal.StaticMesh):
        generate_collision(asset, "convex")
